# MahJong Writer Content Creation Workflow

Agent Instructions for Blog Post & Social Content Creation

---

## Ideal Customer Profile (ICP)

Every piece of content created with this workflow is written for **one specific reader**. Keep her in mind at every phase.

**Primary Audience: Women, 30–50 years old**

| Dimension | Profile |
|-----------|---------|
| Age | 30–50 |
| Gender | Primarily women |
| Interests | Spirituality, self-growth, relationships, intuition, Eastern wisdom |
| Relationship to astrology | Curious to deeply engaged — may know Western astrology but is new to or exploring Chinese Astrology |
| Emotional state | Seeking clarity, guidance, and meaning — especially around love, relationships, career, and life transitions |
| Values | Authenticity, ancient wisdom, personal empowerment, cultural richness |
| Reading habits | Mobile-first, skims on first pass, reads deeply when a headline or opening hooks her |
| Language | Plain, warm, direct — not academic, not preachy, never condescending |

**Write as if you are speaking to one woman** — not a crowd. She is thoughtful, spiritually open, and looking for guidance she can trust and act on.

## Phase 1: Ideation & Format Selection

### Step 1.1: Accept the Idea
* Human provides: blog topic, angle, or brief
* Agent action: Acknowledge receipt of the idea
* Move to 1.2

### Step 1.2: Ask Clarifying Questions
Before format selection, gather only what's needed to write with the right voice and angle. Keep it to 2 questions maximum — focused entirely on the Chinese Astrology topic and reader experience:
* **Reader moment** — Where is our reader likely to be in her life right now in relation to this topic? (e.g., navigating a relationship, seeking career clarity, going through a life transition, exploring her spiritual path)
* **Emotional angle** — What tone best serves her right now: warm and hopeful, mystical and intuitive, or practical and empowering?

Agent action: Ask only these 2 questions in a conversational single message. Do NOT ask about business goals, external sources, or references — source material will be provided separately. Move to 1.3.

### Step 1.3: Confirm Understanding
Agent action: In 2–3 sentences, summarize the topic, the reader moment, and the emotional angle as you understand them. Ask: "Does this capture what you have in mind, or should we adjust before moving forward?" Wait for confirmation before moving to Phase 2.

## Source Material Policy

> ⚠️ **This is a non-negotiable rule for all content created with this workflow.**

All Chinese Astrology content — including sign meanings, elemental interpretations, compatibility analysis, monthly and yearly forecasts, and any spiritual guidance — must be drawn **exclusively from the source material provided by the MahJongTarot.com team.**

**Never:**
* Invent or improvise sign meanings or interpretations
* Draw from external astrology websites, books, or general knowledge
* Blend outside astrology systems (Western, Vedic, etc.) unless explicitly instructed

**The only permitted external augmentation:**
* Current events relevant to the topic
* Upcoming holidays, festivals, or culturally significant dates
* Seasonal or calendar context that makes the post timely

When source material has not yet been provided, the agent should note: *"Source material needed before drafting — please share your reference files."*

**Source material location:** `content/sources/` — All reference files (sign meanings, elemental interpretations, compatibility charts, etc.) should be stored here and referenced during drafting.

## Phase 2: Format & Style Selection

### Step 2.1: Recommend Blog Format + Alternatives
Based on the idea, recommend the single best blog type and explain why it's the right fit.

**MahJong Writer Blog Type Catalogue:**

1. **The Origin Story** — Shares a personal journey or transformation to build trust and connection (e.g., "How Understanding My Chinese Zodiac Sign Changed How I Approach Love")
2. **The Myth-Buster** — Challenges a widely-held belief the audience holds, with evidence and reframe (e.g., "Why Your Western Horoscope Isn't Telling You the Whole Story About Love")
3. **Round-Up / Expert Voices** — Aggregates tips, quotes, or perspectives from multiple credible sources (e.g., "What Each Chinese Zodiac Sign Should Know About Love This May")
4. **Before & After Framework** — Contrasts old thinking or behavior with new, showing the transformation clearly (e.g., "Old Approach vs. New Awareness: How Chinese Astrology Changes How You Date")
5. **The Curated List** — A high-value, tightly edited collection of insights, signs, or guidance (e.g., "The 5 Chinese Zodiac Signs Most Favored for Love This May")

> **Recommended:** [Format Name]
> **Why:** [1-2 sentences on why this format suits the idea]
> **Alternatives to consider:**
> * [Format 2]
> * [Format 3]
> * [Format 4]

Wait for human confirmation. Move to 2.2.

* Main sections (3-5 sections typical)
* Key takeaways per section
* Closing statement / Call-to-action direction

Agent action: Present outline. Ask: "Does this structure work, or should we adjust?"

### Step 3.2: Source Material Check
Before writing, confirm source material is available in `content/sources/` for this topic. If found, note which files you'll draw from. If not found, pause and ask: "I don't see source material for this topic in content/sources/ — please share the reference files before I begin drafting." Do not proceed to Phase 4 without source material.

## Phase 4: First Draft in Chat

### Step 4.1: Write Blog Draft
Using the approved outline and confirmed blog format, write the full English blog post (`blog.md`) inline in the chat.

**Writing guidelines:**
* Length: 800–1500 words (flexible if topic demands more)
* Headings: Break up sections so a reader scanning on mobile gets the gist from headings alone
* Hook: Open with a sentence that makes the reader feel seen or intrigued ("If you've ever wondered..." / "The ancient wisdom of Chinese Astrology says...")
* Close: End with a clear takeaway or next step—never a vague note
* Tone: Warm, mystical, and empowering — write for a woman aged 30–50 who is spiritually open, seeking real guidance (not vague affirmations), and wants to understand how Chinese Astrology applies to her actual life right now — her relationships, her decisions, her sense of purpose
* **Source rule:** All Chinese Astrology content must come from the provided source material. Never invent interpretations, invent sign meanings, or draw from outside references. External sources may only be used to add current event context, upcoming holidays, or relevant dates.

Agent action:
* Write the full draft in chat
* At the end, ask: "Does this work, or what should I revise?"

Move to Phase 5 (Iteration) if revisions needed, or Phase 6 (Approval) if human is satisfied.

## Phase 6: SEO Metadata

### Step 6.1: Analyze for SEO Elements
While the draft is fresh, identify:
* **Primary keyword:** The 1-3 word phrase this post ranks for (e.g., "Chinese astrology love", "zodiac romance May")
* **Secondary keywords:** 2-3 related phrases
* **Slug:** URL-safe version of title (lowercase, hyphens, e.g., `chinese-astrology-love-may`)
* **Meta title:** 50–60 characters, includes primary keyword, compelling (not just the blog title)
* **Meta description:** 155–160 characters, summarizes the post and includes primary keyword, drives clicks from search

Agent action:
* Present SEO elements you've derived from the approved draft
* Ask: "Do these keywords and descriptions feel right, or should we adjust?"
* Wait for confirmation before moving to Phase 7

Once confirmed, move to Phase 7.

## Phase 7: Additional Content Formats

### Step 7.1: Vietnamese Translation
Translate the approved `blog.md` into fluent, natural Vietnamese.

**Guidelines:**
* Natural, fluent Vietnamese — not stiff machine-translate tone
* Our Vietnamese reader shares the same ICP profile: women 30–50, spiritually curious, seeking guidance around love, relationships, and life direction
* Preserve the warmth and mystical tone of the English original — do not flatten it into neutral language
* Chinese Astrology terms (sign names, element names, etc.) should use the standard Vietnamese equivalents where they exist (e.g., Tuổi Tý, Ngũ Hành, etc.)
* Maintain the same structure and headings as the English version

Agent action: Create `vietnamese.md` with full translation.

### Step 7.2: Circle.so Post
Format the approved blog content for publishing to the Circle.so community platform, where our spiritually-curious audience gathers to learn, share, and discuss Chinese Astrology and personal growth.

**Important:** Circle.so does not support raw HTML or custom CSS. This must be plain, structured content written for Circle's native rich text editor.

**Guidelines:**
* Opening hook paragraph — 2–3 sentences, warm and inviting
* Use Circle's native heading levels (H2, H3) for sections
* Keep the same core content as `blog.md` but simplify structure for community context
* Add a brief intro note: "This post is from the MahjongTarot.com blog. Full version at [link]."
* Close with a discussion prompt to encourage community engagement (e.g., "Which zodiac sign resonates most with you this month?")
* No custom HTML, no inline CSS, no special formatting beyond bold/italic/headings/bullets

Agent action: Create `circle-post.md` — a clean markdown version formatted for copy-paste into Circle's editor.

### Step 7.3: Social Posts
Write all social media posts based on the approved blog draft. Create `social.md` with posts for:

> **Our ICP (women 30–50, spiritually curious) lives on Instagram and Pinterest — not LinkedIn. Never write a LinkedIn post for MahjongTarot content.**

**Instagram Caption**
* Opening line must stop the scroll — make her feel seen or deeply intrigued
* Warm, mystical, personal tone — write as one woman speaking to another
* 3–5 short paragraphs or staccato lines with generous spacing
* End with a soft CTA: "Link in bio ✨" or a reflective question that invites comments
* 8–15 hashtags on a new line after the caption, focused on: Chinese Astrology, zodiac, spirituality, self-growth, relationships, women's wellness
* Example hashtags: #ChineseAstrology #ChineseZodiac #ZodiacSigns #SpiritualGrowth #AncientWisdom #WomensWellness #AstrologyLovers #FireHorse2026 #SelfDiscovery #MindBodySpirit

**Pinterest Pin Description**
* 150–300 characters — keyword-rich for Pinterest search, not hashtag-heavy
* Describe what the reader will learn or gain from clicking
* Lead with the topic keyword naturally (e.g., "Chinese Astrology reveals…")
* Warm but informational tone — Pinterest is a discovery engine, not a social feed
* 2–3 hashtags maximum (Pinterest rewards keywords over hashtags)
* Example: "Chinese Astrology reveals how your zodiac element shapes the way you heal from heartbreak — Wood, Fire, Earth, Metal, Water each grieve differently. Find your element. #ChineseAstrology #ZodiacHealing"

**Facebook Post**
* 2–4 sentences, conversational and warm
* Speaks to women who follow astrology and seek spiritual guidance
* Personal, not promotional — write as if sharing something meaningful with a friend
* End with "[Link in first comment]"

**Twitter / X Post**
* ≤280 characters
* Lead with a bold insight, intriguing question, or surprising Chinese Astrology fact
* Warm and direct — not corporate, not clever-for-clever's-sake

**Short Email Subject Line**
* 40–50 characters
* Curiosity-driven or emotionally resonant — speak directly to her life situation

Agent action: Draft all social posts and present in `social.md` format. Ask for approval.

Once approved, move to Phase 8.

> **Platform clarity:**
> * `blog.md` → mahjongtarot.com (public blog)
> * `circle-post.md` → Circle.so community (plain text, paste into Circle editor)
> * `social.md` → Instagram, Pinterest, Facebook, Twitter/X, Email

## Phase 8: Image Generation (Nano Banana 2 via Browser Automation)

> **Image model:** Nano Banana 2 — Google's Gemini image generation model
> **Method:** Browser automation via Claude in Chrome (no API key required)
> **Skill:** `generate-image` — see `agents/writer/context/skills/generate-image/SKILL.md`

### Step 8.1: Create Image Prompt
Using the confirmed image style (from Phase 2.2) and the blog topic, craft a natural language image prompt and present it for approval before generating.

**Prompt construction guidelines:**
* Lead with the image style (e.g., "Elemental Drama style:")
* Name the primary subject clearly (zodiac animal, element, symbolic scene)
* Specify mood, lighting, and color palette aligned with brand tokens:
  - Midnight Indigo: `#1B1F3B` — backgrounds, shadow areas
  - Mystic Fire: `#C0392B` — fire elements, energy accents
  - Celestial Gold: `#C9A84C` — light, highlights, celestial glow
  - Warm Cream: `#FAF8F4` — mist, ethereal light
* Include composition direction (foreground/background, focal point)
* End with exclusions — what NOT to include (Western astrology symbols, generic stock photography, text overlays, rounded shapes)
* Always 16:9 aspect ratio for blog hero images

**Zodiac visual language:**
* Fire signs: warm amber, orange flames, dynamic motion
* Water signs: deep teal, moonlight, fluid forms
* Earth signs: rich brown, stone, grounded and still
* Metal signs: silver, sharp geometry, clean lines
* Wood signs: jade green, organic growth, branching forms

**Example prompt:**
```
Elemental Drama style: A powerful horse galloping through a sea of crimson
and gold fire, mane blazing with celestial energy, sharp geometric sparks
trailing behind. Deep navy-black background (#1B1F3B) with dramatic
top-lighting in gold (#C9A84C) and crimson (#C0392B). Cinematic depth of
field, horse fills the center-right of the frame, flames sweep left to right.
Aspect ratio: 16:9. Exclude: Western zodiac symbols, text, watermarks,
anime style, rounded shapes, soft pastel colors.
```

Agent action:
* Write the prompt
* Present it and ask: "Should I send this to Nano Banana to generate the image?"
* Wait for confirmation before generating

### Step 8.2: Generate Image via Browser Automation

Once human confirms, invoke the `generate-image` skill which handles:

1. **Open Gemini** — Navigate to `https://gemini.google.com/app` in Claude in Chrome
2. **Enter the prompt** — Type the approved prompt into Gemini's chat input
3. **Wait for generation** — Nano Banana generates the image (10–30 seconds)
4. **Download the image** — Save the generated image (with user permission)
5. **Optimize to WebP** — Resize to 1200×630, convert to WebP at quality 82, save to `website/public/images/blog/[slug].webp`
6. **Archive the original** — Save the PNG to `content/topics/[topic-slug]/[slug]-hero.png`

**If Gemini requires sign-in:** Stop and ask the user to log into their Google account in the browser, then resume.

**If generation fails:** Screenshot the error, simplify the prompt (remove hex codes, simplify description), retry once. If still failing, report to user.

**If the image has unwanted text/watermarks:** Regenerate with stronger exclusion: "Absolutely no text, words, letters, or watermarks anywhere in the image."

Agent action:
* Run the `generate-image` skill with the approved prompt
* Confirm: "Image generated and saved to both `content/topics/[topic-slug]/` and `website/public/images/blog/`"
* If user is not satisfied, adjust the prompt and regenerate
* Move to Phase 9

## Phase 10: Publish to Website

### Step 10.1: Deploy Blog Post to Website
Once the Phase 9 checklist passes and the human confirms, publish the post to the live website by copying the final assets into the website directory and pushing to GitHub.

**Deployment steps:**

1. **Copy blog post to website:**
   * Copy `content/topics/[topic-slug]/blog.md` → `website/blog/posts/[topic-slug].md`
   * Copy hero image `content/topics/[topic-slug]/[image-name].png` → `website/blog/images/[topic-slug]-hero.png`

2. **Stage, commit, and push to GitHub:**
   ```bash
   git add MahjongMirror/content/topics/[topic-slug]/
   git add MahjongMirror/website/blog/posts/[topic-slug].md
   git add MahjongMirror/website/blog/images/[topic-slug]-hero.png
   git commit -m "Publish blog post: [topic-slug]

   - Blog post (EN): content/topics/[topic-slug]/blog.md
   - Vietnamese translation: content/topics/[topic-slug]/vietnamese.md
   - Social posts: content/topics/[topic-slug]/social.md
   - Circle.so post: content/topics/[topic-slug]/circle-post.md
   - SEO metadata: content/topics/[topic-slug]/seo.md
   - Styled HTML page: content/topics/[topic-slug]/post.html
   - Hero image: website/blog/images/[topic-slug]-hero.png"
   git push origin main
   ```

3. **Verify push succeeded:**
   * Confirm the push completed without errors
   * If push fails due to authentication, advise human to check GitHub credentials

Agent action:
* Execute deployment steps
* Confirm: "✅ Blog post published and pushed to GitHub repository"
* If any step fails, report the error and suggest a fix
