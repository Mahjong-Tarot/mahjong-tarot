Local Machine Setup
Cowork Code 20–30 min

Claude takes the wheel. Stage A uses Cowork (screen control) to install developer tools. Stage B runs in Code to scaffold your project and configure Claude itself.

Install git, node, Vercel CLI, Supabase CLI. Authenticate GitHub. Pre-seed all Claude Code permissions.
Stage B — Code
Create project folder + git repo. Write CLAUDE.md rules. Configure Supabase MCP. Install skills (skill-creator, agent-creator).

P2
Website Infrastructure + Core Agent Setup
Code 45–60 min

No interview. Claude scaffolds your website with placeholder content, installs all 4 core agents from a proven template, and auto-registers the PM's daily/weekly schedules. Your team is running before you've answered a single business question.

2 questions
Project name + timezone. That's all Claude needs to begin.
Website
Scaffold Next.js → push to GitHub → deploy to Vercel → wire Supabase. Placeholder content throughout.
Core agents ×4
Project Manager · Writer · Designer · Web Developer installed from template.
PM schedules
4 tasks auto-registered via MCP: 7am reminder, 9am compile, 5pm EOD, Friday RAG report.
Outputs
Live website on Vercel Supabase connected 4 agents installed 4 PM schedules active .claude/agents/*.md agents/*/context/persona.md

Extended Team Schedules + Verification
Code Chat 30–45 min

PM schedules are already active from P2. This phase handles schedules for any specialist agents added in P3, then runs four end-to-end verification tests to confirm the whole system is working.

Section 1 — Code
Generate /schedule commands for extended agents (Product Manager, Marketing Manager, Social Media Manager — if installed).
Section 2 — Chat
Paste each command into the Chat tab to activate. Skip entirely if only core agents were installed.
Section 3 — Code
4 verification tests: file checks, PM standup dry-run, Writer caption, Web Dev build-page, Designer prompt.
