Infrastructure 📦

GitHub

Your project's shared filing cabinet with infinite undo. Every file, every change, every version — permanently stored and retrievable.

What it stores
Your entire project: website code, agent instruction files, brand guides, blog drafts, and config. Everything in one place, accessible from any computer.
Why it matters
Every change your AI team makes is recorded with a timestamp and description. If something breaks, you can rewind to any previous state — like Command+Z that never expires.
How your AI uses it
When the Web Developer publishes a blog post, it pushes the change to GitHub first. Vercel (your website host) sees the change and automatically updates your live site.
What you do
Create a free account. That's it. Claude handles all the git commands — pushing, pulling, branching — you never need to touch the terminal.
Technical name: git repository hosting · Free tier is sufficient
Infrastructure 🌐

Vercel

Your website's home address on the internet. It watches GitHub, and every time your AI team makes an update, your live site refreshes automatically.

⚡
The magic: push to GitHub → website updates itself
You never FTP files, log into a hosting panel, or manually upload anything. Your AI team commits to GitHub → Vercel detects the commit → your site is live in ~90 seconds.
What Vercel hosts
Your Next.js website — all pages, the blog, the contact form, and the newsletter signup. It handles traffic, performance, and uptime automatically.
How you connect it
Sign up with GitHub (one click), import your repo, set the project root to website/, and deploy. That's the entire setup.
Your URL
You start with a free yoursite.vercel.app URL. Connect a custom domain at any time — Vercel handles SSL certificates automatically.
Technical name: serverless hosting / CDN · Free tier covers most small businesses
Your Data 🗄️

Supabase

Your customer records system and backend office. Every form submission, newsletter signup, and contact inquiry is stored here — and your AI agents can read it directly.

What it stores
Contact form submissions, newsletter signups, reading inquiries, and any other data your website collects. Stored in a proper database — not a spreadsheet.
Why it's separate from the website
Your website is the front door. Supabase is the back office. Keeping them separate means your data is safe even if you change your website completely.
How Claude connects to it
Via MCP (explained in the next slide). Once configured, Claude can query your database directly — no copy-pasting, no exports, no manual lookups.
What you set up
Create a project, copy two values (URL + API key), paste them into Vercel. Claude handles the database schema (the table structure) automatically in P2.
Technical name: PostgreSQL database + backend-as-a-service · Free tier: 500MB storage

AI Infrastructure 🔌

MCP — Claude's Universal Adapter

Think of MCP like a universal power adapter. It's a standard plug that lets Claude connect to any external service — database, calendar, email — without custom code for each one.

Without MCP
You'd have to copy data from Supabase, paste it into Claude, then copy Claude's answer back into Supabase. Every. Single. Time.
With MCP
Claude asks your database a question directly — just like a human employee logging into your CRM. No copy-pasting. Real-time data. Automatic.
MCP: Supabase
Claude reads and writes to your customer database — contact forms, subscriber lists, booking requests.
MCP: Context mode
Keeps Claude's working memory efficient on large codebases — like a librarian who indexes books instead of reading them all upfront.
MCP: Browser
Lets Claude open a browser, take screenshots, and test your website — the way a QA tester would.
Technical name: Model Context Protocol — open standard by Anthropic · Configured in .claude/settings.local.json

Your AI Team 📖

Skills

Your team's standard operating procedures — written down so Claude follows the exact same process every time, not whatever it feels like doing that day.

What a skill is
A markdown file that describes a specific task step-by-step: what to ask, what to check, how to format the output, what to do if something goes wrong. Like a recipe, not a suggestion.
Why they matter
Without skills, Claude improvises. With skills, every standup looks the same, every RAID entry has the right columns, every blog post follows the same structure. Consistency without micromanaging.
daily-checkin
Guides the team through a standup and saves the result to the right file.
raid-log
Logs risks, issues, and dependencies in the standard format with proper columns.
scope-change
Produces a formal change request with impact assessment before any decision.
prioritise
Scores any idea with RICE before adding it to the roadmap.
Skills live in: ~/.claude/skills/{name}/SKILL.md (global) and .claude/skills/ (project) · Invoked with /skill-name

Your AI Workspace 🔑

Permissions — Trusting Claude to Work

By default, Claude asks for your approval before running any terminal command. During bootstrap, this creates constant interruptions. We pre-approve everything once so it never happens again.

What we set up
A single Bash(*) permission that pre-approves all terminal commands — instead of approving each tool individually every time it's used.
What stays protected
File writes still ask for your confirmation. Claude can run commands freely, but creating or editing files still shows you a preview before saving. You keep oversight over content.
Before setup
Every `git push`, every `npm install`, every database command → a permission popup. Dozens per session.
After setup
Claude runs commands freely. Scheduled agents fire at 7am and 9am without waking you up for approvals.
Where it lives
~/.claude/settings.local.json — configured once in P1, applies to every project and every scheduled task forever.

Project Management SOPs 📊

How Your Project Manager Works

Your PM agent follows industry-standard professional practice — the same formats used in corporate project management, applied automatically without setup from you.

📝 Daily Standup (Scrum)
Three questions per person: What did you do yesterday? What are you doing today? Any blockers? Compiled into a team briefing at 9am. Missing check-ins noted as "Not received" — never fabricated.
⚠️ RAID Log
Risks · Assumptions · Issues · Dependencies. Every problem gets logged with: ID, probability, impact score, owner, and action. Append-only — nothing is ever deleted.
📈 RAG Status Report
Weekly health check using 🟢 Green (on track), 🟡 Amber (at risk), 🔴 Red (blocked). Automatically generated every Friday from the week's standups and open RAID items.
📋
Scope changes always go through a formal change request
Any new work, feature, or requirement gets a CR document: description, business case, impact on timeline and budget, three options (Accept/Reject/Defer). Your PM recommends. You decide.
Product Management SOPs 🗺️

How Your Product Manager Works

Your Product Manager applies the same frameworks used by product teams at growth-stage companies — adapted for a solo business owner who needs results without complexity.

🎯 OKRs (quarterly goals)
Objective (the qualitative goal: "Become the go-to resource for X") + Key Results (the numbers: "500 newsletter subscribers, 12 blog posts, 3 speaking opportunities"). Reviewed quarterly. Never changed mid-quarter without sign-off.
🧮 RICE Prioritisation
Every feature request is scored: Reach × Impact × Confidence ÷ Effort. Score >100 = do now. 50–100 = plan for next quarter. <50 = park it. No gut-feel decisions.
🗓️ Now/Next/Later Roadmap
Now: max 3 things this quarter. Next: planned but not started. Later: ideas with no commitment. Entries describe outcomes, not features.
🔍 Competitive Analysis
Updated monthly using live research. Feature comparison matrix + strategic insights. Flags competitor moves that affect your roadmap.
⭐ North Star Metric
One number that captures the core value you deliver — derived from your 90-day goal. Everything the Product Manager does connects back to moving this number.
