## Why phased and not parallel

The three load-bearing assumptions in [`product.md`](./product.md) §7 each need to be tested against reality before the next bet is funded:

1. **Phase 1 tests assumption #2** (almanac is a wedge, not a niche). If D7 < 20%, we stop and re-examine.
2. **Phase 2 tests assumption #1** (audience exists at price). If paid acquisition CTR < 1.5% / conversion < 0.5%, we stop.
3. **Phase 3 tests assumption #3** (voice scales without Bill personally writing). If blind taste ≠ 50/50, we stop the content engine.

Phases 4–6 only make sense if 1–3 pass. We do not build ahead of the validation gate.
